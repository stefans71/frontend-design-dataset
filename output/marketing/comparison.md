## Key Patterns

**Prompt adherence:** The fine-tuned model follows multi-constraint prompts more literally — it doesn't add unrequested UI chrome (carousels, social feeds, extra nav panels) and doesn't omit requested elements (tier comparison, active states, accent colors).

**Brand specificity:** Given a brand name and domain, the fine-tuned model generates contextually appropriate copy and CTAs. The base model tends toward generic placeholder text and SaaS defaults regardless of the specified context.

**Color constraint fidelity:** When a specific accent color is named (purple, green, orange), the fine-tuned model applies it consistently across all interactive elements. The base model frequently defaults to blue regardless of the prompt.

**Layout complexity:** The fine-tuned model handles multi-component layouts (3-tier pricing, dual KPI cards) correctly on first generation. The base model tends to simplify to a single unit.
